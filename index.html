MFL.football

A responsive my fantasy league template from MFLaddons.

View demo

Template instructions

The template will make use of Home Page Messages. You may have already used these from previous years. Save any code you need from these on your computer and delete them from your MFL site before you get started. Also make sure that you don't have one of your HPM's set as a header or footer already as that will interfere with the installation.

If you have any questions with installation or have ideas, please post here.

If you are installing on a new site it may look something like this: If you already named your HPM's it may look a little different. In this example header would be #1, footer #2, homepage #3, message board #4, and so on...

  1. To install the template you'll need to log in as commish and go to the commish setup page.

  2. On the "Select a skin" page choose "none".

  7. Go to the homepage modules and tabs setup page. On the main tab add a new column at 100% and place the homepage HPM directly underneath.

  8. Remove all other modules from the main tab and delete all other tabs. This template will only use the main tab. Multiple tabs aren't supported.

    Once you are done you'll only have the main tab. All other tabs will be removed.
    Do not add any additional modules on this page. Adding modules will now be done by adding a snippet of code into the homepage message we added in step 4. You can find the code to add modules here.

  9. Take a look at your homepage. Then look at a few other pages. If everything look good...

Tricks

For faster mobile-friendly development, use these utility classes for showing and hiding content by device via media query.

Try to use these on a limited basis and avoid creating entirely different versions of the same site. Instead, use them to complement each device's presentation.

A great use case is with the standings. There are 2 standings modules available, brief and standard. The brief one is perfect for mobile while the standard one works great on desktop and tablet.

Example 
<div class="mobile-standings-wrapper hidden-xs">
  <module name="STANDINGS" UNIT="DIVISION00" />
  <module name="STANDINGS" UNIT="DIVISION01" />
</div>
<div class="visible-xs-block">
  <module name="BRIEF_STANDINGS" />
</div>

In this example the brief standings is used on phones, while the standard standings broke out by division is used on desktops and tablets.

Available classes

Use a single or combination of the available classes for toggling content across viewport breakpoints.

Extra small devices Phones (<768px) Small devices Tablets (≥768px) Medium devices Desktops (≥992px) Large devices Desktops (≥1200px)
.visible-xs-* Visible Hidden Hidden Hidden
.visible-sm-* Hidden Visible Hidden Hidden
.visible-md-* Hidden Hidden Visible Hidden
.visible-lg-* Hidden Hidden Hidden Visible
.hidden-xs Hidden Visible Visible Visible
.hidden-sm Visible Hidden Visible Visible
.hidden-md Visible Visible Hidden Visible
.hidden-lg Visible Visible Visible Hidden

The .visible-*-* classes for each breakpoint come in three variations, one for each CSS display property value listed below.

Group of classes CSS display
.visible-*-block display: block;
.visible-*-inline display: inline;
.visible-*-inline-block display: inline-block;

So, for extra small (xs) screens for example, the available .visible-*-* classes are: .visible-xs-block, .visible-xs-inline, and .visible-xs-inline-block.
